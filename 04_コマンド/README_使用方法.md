# 自動添削システム - 起動コマンド

このディレクトリには、自動添削システムの統合起動コマンドファイルが含まれています。

## ファイル一覧

- **自動添削システム.sh** - macOS/Linux用の統合起動スクリプト（ブラウザ自動起動機能付き）
- **自動添削システム.bat** - Windows用の統合起動バッチファイル（ブラウザ自動起動機能付き）

## 使用方法

### 開発サーバーの起動（デフォルト）

**macOS/Linux:**
```bash
./04_コマンド/自動添削システム.sh
```
または
```bash
./04_コマンド/自動添削システム.sh dev
```

**Windows:**
```cmd
04_コマンド\自動添削システム.bat
```
または
```cmd
04_コマンド\自動添削システム.bat dev
```

### ビルド（本番用）

**macOS/Linux:**
```bash
./04_コマンド/自動添削システム.sh build
```

**Windows:**
```cmd
04_コマンド\自動添削システム.bat build
```

### 本番サーバーの起動

**macOS/Linux:**
```bash
./04_コマンド/自動添削システム.sh start
```

**Windows:**
```cmd
04_コマンド\自動添削システム.bat start
```

## コマンド一覧

| コマンド | 説明 |
|---------|------|
| `dev` | 開発サーバーを起動します（デフォルト） |
| `build` | 本番用にビルドします |
| `start` | 本番サーバーを起動します（事前にビルドが必要） |

## 直接実行する場合

ターミナルで直接実行することもできます：
```bash
cd web
npm run dev    # 開発サーバー
npm run build  # ビルド
npm start      # 本番サーバー
```

## 機能

- **ブラウザ自動起動**: サーバー起動後、自動的にブラウザが開きます
- **ポート自動解放**: ポート3000が使用中の場合は、既存プロセスを自動停止します
- **依存関係自動インストール**: 初回起動時に`node_modules`がない場合は自動インストールします

## 注意事項

- 初回起動時は、依存関係が自動的にインストールされます
- `.env.local`ファイルに環境変数（GEMINI_API_KEYなど）を設定してください
- 開発サーバーは通常 `http://localhost:3000` で起動します
- サーバー起動後、自動的にブラウザが開きます（最大15秒待機）

## トラブルシューティング

### ポートが既に使用されている場合

別のポートで起動する場合：
```bash
cd web
PORT=3001 npm run dev
```

### 権限エラー（macOS/Linux）

スクリプトに実行権限を付与：
```bash
chmod +x 04_コマンド/*.sh
```
