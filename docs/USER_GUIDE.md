## 1. はじめに

### EduShiftとは？

EduShiftは、中学受験・高校受験の国語記述問題を自動採点するAIシステムです。

**特徴：**
- ✅ 手書き答案をAIが読み取り（OCR）
- ✅ 模範解答と照合して採点
- ✅ 良い点・改善点・書き直し例を提示
- ✅ 採点の厳しさを3段階で調整可能
- ✅ 複数生徒の一括採点に対応

### 採点の5大原則

EduShiftは以下の5つの観点で採点を行います：

| 観点 | 説明 |
|------|------|
| **1. 設問要求** | 「〜とはどういうことか」「なぜか」など設問の要求に合っているか |
| **2. 本文根拠** | 本文の内容に基づいた解答になっているか |
| **3. 指定条件** | 字数制限、キーワード、「本文中の言葉を使って」などの条件を満たしているか |
| **4. 減点ルール** | 誤字脱字、主述のねじれ、文末表現の誤りなど |
| **5. 文章形式** | 句読点、接続詞、文の構成が適切か |

## 2. 採点を始める

### Step 1: ファイルの準備

以下の2つのファイルを用意してください：

| ファイル | 内容 | 例 |
|----------|------|-----|
| **生徒答案** | 生徒が書いた解答の写真・スキャン | 手書きノート、解答用紙の写真 |
| **模範解答** | 問題文と模範解答が載っている資料 | 問題集の解答ページ、塾のプリント |

### Step 2: アップロード

1. 「生徒の解答をアップロード」欄に**生徒答案**をドラッグ＆ドロップ
2. 「解答・問題画像をアップロード」欄に**模範解答**をドラッグ＆ドロップ

> 💡 **スマホをお使いの場合**: アップロード欄をタップするとカメラが起動します。その場で答案を撮影し、そのままアップロードできます。撮影した画像は自動的に圧縮されるため、高画質のまま撮影して大丈夫です。

### Step 3: 問題番号の入力

採点したい問題番号を入力します。

**入力例：**
- `問1`
- `問2-1`
- `(1)`
- `(2)-ア`
- `大問3 問2`

> 💡 **ヒント**: 模範解答の問題番号と同じ形式で入力すると、AIが正確に問題を特定できます。

### Step 4: 採点の厳しさを選択（任意）

| レベル | 説明 | おすすめの場面 |
|--------|------|----------------|
| **甘め** | 部分点を多めに付与、表現の揺れを許容 | 初めて記述に挑戦する生徒、モチベーション重視 |
| **標準** | 一般的な入試基準で採点 | 通常の演習、実力確認 |
| **厳しめ** | 細かな条件も厳格に判定 | 入試直前期、本番想定の演習 |

### Step 5: 採点実行

「採点する」ボタンをクリックして、結果を待ちます（通常10〜30秒）。

## 3. 問題の保存と読み込み

同じ問題で繰り返し採点する場合、問題設定を保存しておくと便利です。

### 保存する

1. 採点画面で問題ファイル・模範解答・問題番号などを設定した状態で「問題を保存」をクリック
2. わかりやすい問題名をつけて保存（例：「2025年 開成中 大問2 問3」）

### 保存される内容

| 保存される | 保存されない |
|-----------|-------------|
| 問題ラベル（問題番号） | 生徒の答案ファイル |
| 配点・採点形式 | |
| 問題ファイル・模範解答ファイル | |
| 模範解答テキスト | |

### 保存した問題を使う

保存済みの問題一覧から選択すると、保存した設定が自動で読み込まれます。あとは生徒の答案をアップロードするだけで採点できます。

> ⚠️ **ご注意**: 保存先はお使いのブラウザ内です。別のブラウザやデバイスからは利用できません。また、ブラウザのデータを消去すると保存した問題も削除されます。

## 4. 採点結果の見方

### 採点レポートの構成

```
┌─────────────────────────────────────────┐
│  得点: 8/10点                            │
├─────────────────────────────────────────┤
│  📝 認識されたテキスト                    │
│  「主人公は〜と感じていた。」              │
├─────────────────────────────────────────┤
│  ✅ 良い点                               │
│  ・本文の表現を適切に引用している          │
│  ・設問の要求に沿った解答形式             │
├─────────────────────────────────────────┤
│  📈 改善アドバイス                        │
│  ・「なぜなら」の根拠がやや弱い            │
│  ・結論部分をより具体的に                  │
├─────────────────────────────────────────┤
│  ✏️ 書き直し例                           │
│  「主人公は〜という経験から、              │
│   〜と感じていたと考えられる。」           │
├─────────────────────────────────────────┤
│  📉 減点理由                             │
│  ・根拠の不足 (-2点)                      │
└─────────────────────────────────────────┘
```

### 各項目の活用法

| 項目 | 活用法 |
|------|--------|
| **得点** | 現在の実力把握、目標点との差を確認 |
| **認識テキスト** | AIが正しく読み取れているか確認（誤読があれば再撮影） |
| **良い点** | 生徒へのフィードバック、継続すべきポイント |
| **改善アドバイス** | 次回の記述で意識するポイント |
| **書き直し例** | 模範的な解答の書き方を学習 |
| **減点理由** | 具体的な失点原因を把握 |

## 5. 再採点する

採点結果に納得がいかない場合や、読み取り結果に誤りがあった場合は、再採点ができます。

### 再採点の方法

採点結果の画面に表示される以下のボタンから再採点できます：

| ボタン | 説明 |
|--------|------|
| **「もっと厳しくで再採点」** | より厳格な基準で採点し直します |
| **「もっと甘くで再採点」** | より寛容な基準で採点し直します |

### AI読み取り結果の修正

AIの文字読み取り結果に誤りがある場合は、認識テキストを手動で修正してから再採点することもできます。

### 再採点のルール

- 初回採点後、**無料で2回まで**再採点できます
- 再採点では**採点回数は消費されません**（追加料金なし）
- 再採点は採点後**7日以内**に行ってください

> 💡 **ヒント**: まず「標準」で採点して結果を確認し、必要に応じて厳しさを変えて再採点するのがおすすめです。

## 6. 一括採点（バッチ処理）

### 複数生徒の答案を一度に採点

1. メイン画面で「一括採点」タブを選択
2. 「生徒を追加」ボタンで生徒カードを追加（最大10名分）
3. 各生徒カードに名前を入力し、答案ファイルをアップロード（1生徒あたり最大5ファイル）
4. 共通の模範解答をアップロード
5. 問題番号と採点基準を設定
6. 「一括採点開始」をクリック

### 生徒カードの操作

| 操作 | 方法 |
|------|------|
| **生徒の追加** | 「生徒を追加」ボタンをクリック |
| **名前の設定** | 各カードの名前欄に生徒名を入力 |
| **答案のアップロード** | 各カードのアップロード欄にファイルをドラッグ＆ドロップ |
| **生徒の削除** | 各カードの削除ボタンをクリック |

### 進捗確認

- 処理中は進捗バーで状況を確認できます
- 各生徒の採点完了後、順次結果が表示されます

### 結果のダウンロード

一括採点が完了すると、全生徒のPDFレポートをまとめたZIPファイルをダウンロードできます。

## 8. アカウント管理

### ログイン情報

- メールアドレスとパスワードでログイン
- パスワードを忘れた場合は「パスワードをお忘れですか？」からリセット

### デバイス制限について

**1アカウントにつき2台まで**のデバイスで利用可能です。

3台目のデバイスでログインしようとすると：
1. 「デバイス上限に達しました」というメッセージが表示
2. 既存デバイスの一覧が表示
3. 不要なデバイスを選択して削除
4. 新しいデバイスでログイン可能に

> 💡 **なぜ制限があるの？**: 教育機関でのアカウント不正共有を防ぎ、公正なサービス提供を維持するためです。

### 使用状況の確認

「使用状況」ページで以下を確認できます：
- 今月の採点回数
- 残り利用可能回数
- プラン更新日

## 9. 推奨環境・ファイル形式

### 対応ブラウザ

| ブラウザ | 対応状況 |
|----------|----------|
| Chrome | ✅ 推奨 |
| Safari | ✅ 対応 |
| Firefox | ✅ 対応 |
| Edge | ✅ 対応 |

### 対応ファイル形式

| 形式 | 拡張子 | 備考 |
|------|--------|------|
| JPEG | .jpg, .jpeg | 推奨 |
| PNG | .png | 推奨 |
| PDF | .pdf | 複数ページ対応 |
| HEIC | .heic | iPhone写真に対応 |
| GIF | .gif | 対応 |
| WebP | .webp | 対応 |
| BMP | .bmp | 対応 |
| HEIF | .heif | 対応 |

### ファイルサイズ

- **最大**: 50MB/ファイル（画像は自動圧縮されます）
- **推奨**: 2MB以下（処理速度向上のため）

> 💡 **自動圧縮について**: アップロードされた画像は自動的に最適化されます。文字認識に必要な品質は保たれるよう調整されているため、スマホで撮影した高画質の写真でもそのままアップロードしてください。文字の多い答案では高品質モード、それ以外では通常モードが自動で適用されます。

### 撮影のコツ

📸 **きれいに撮影するポイント**：

1. **明るい場所で撮影** - 自然光または明るい照明の下で
2. **真上から撮影** - 斜めからだと文字が歪む
3. **影を避ける** - 手や機器の影が入らないように
4. **全体を収める** - 答案全体がフレーム内に入るように
5. **ピントを合わせる** - 文字がくっきり見えるまで待つ

> 💡 **スキャナーがある場合**: スキャン画像の方がOCR精度が高くなります（推奨解像度: 300dpi）
