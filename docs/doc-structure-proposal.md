## 設計方針

### 対象読者
- **学習塾の講師**: 複数生徒の一括採点、業務効率化が関心事
- **保護者**: 料金・安全性・効果が関心事。技術に不慣れな方も多い
- **中学・高校受験生**: スマホ操作が中心、直感的な説明が必要

### 基本原則
1. **正確性**: コードの実装値に厳密に一致させる
2. **平易さ**: 技術用語は使わず、日常語で説明する
3. **検索性**: 困ったときにすぐ該当箇所を見つけられる構成にする
4. **一貫性**: FAQ、使い方ガイド、料金ページ（アプリ内）の情報を統一する

## 注意: 使い方ページ（usage/page.tsx）との整合

アプリ内の使い方ページ（`/usage`）には「1ファイルあたり4MB以下、合計4MB以下」と記載されている。これは圧縮後のサーバー送信サイズの制限であり、ユーザーがアップロードするファイルの制限（50MB）とは異なる。

ドキュメントではユーザー目線で「50MBまでOK、自動圧縮される」と説明し、技術的な内部制限（4.2MB/4.3MB）には言及しない方針とする。アプリ内の使い方ページも同様に修正することを推奨する（ただし今回のスコープ外）。
